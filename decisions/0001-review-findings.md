# 0001: 战略文档首轮独立审查发现

Date: 2026-05-15（裁决：2026-05-17）
Status: accepted

## Context
首版 8 份战略文档（README/VISION/DIFFERENTIATION/COLLABORATION/ARCHITECTURE/ROADMAP/FEEDBACK_LOOP/CLAUDE.md）由 Cowork 起草后，邀请了一个独立的 Plan 子 Agent 以"刚加入项目的合伙人"视角做冷读。

## 主要发现（按重要性排序）

### 🔴 必须改（影响项目根基）

**F1. "采纳率"作为 MVP 指标违反 VISION §4.6 的"父母赋能"原则**
- 出处：[VISION §4.6] vs [ROADMAP M3.3]
- 问题：把"建议被执行"当成功，会让 prompt 演化为更具指令性
- 处理：把 M3.3 指标从"采纳率"改为"父母讨论/挑战次数"或"父母自己做出新决定的频率"

**F2. 北极星指标过度主观且可能确认偏差**
- 出处：[ROADMAP §0]
- 问题：单用户 + 自己设计 + 自己评判 = 必然达成
- 处理：引入"盲测对照"——同一周事件让通用 prompt（无项目体系）生成洞察，作者盲选哪份更击中；本项目"惊喜率"必须显著高于基线

**F3. "天赋识别"在 2.5 岁 + 单孩子 + 8 周内不可证伪**
- 出处：[ROADMAP §1]
- 问题：学龄前发展信号噪音极高，心理学上对终生天赋几乎无预测效度；样本不足
- 处理：把 MVP 假设降级为"AI 能识别**重复模式**"（可测：本周第 N 次涉及节奏 / 秩序 / 情绪反应模式），把"天赋"留 v2

**F4. 法律/合规章节完全缺失（中国未成年人数据 + 跨境传输 Claude API）**
- 出处：[ARCHITECTURE §8] 缺失
- 问题：《个人信息保护法》《未成年人网络保护条例》对未成年人数据有强制约束，Anthropic API 在境外是数据出境
- 处理：MVP 阶段限定"自用 + 不商业化"，但要在文档里写清楚；v1 商业化前必须完成 ICP + 数据出境评估。新增 ADR-0002

**F5. 跨代复利承诺与"删除即清云端"冲突 + 设备/密钥失效场景**
- 出处：[VISION §6.2] vs [ARCHITECTURE §8]
- 问题：15 年保存承诺没有密钥继承机制；换设备/丢钥匙就报废
- 处理：新增 ADR-0003 "长期保存策略"；密钥托管 / 纸质恢复码 / 家族继承

### 🟡 应该改（可执行性问题）

**F6. 月成本 $5/用户 估算偏乐观**
- 出处：[ARCHITECTURE §6.2]
- 问题：未计入对话往返、A/B 反馈分类自身的 LLM 调用、prompt 迭代成本
- 处理：建一份真实测算 spreadsheet，假设单用户月对话 ~150 次往返；保守估计 $15-25/月，$200 撑 8-12 用户而不是 40-50

**F7. Qwen2.5-3B 中文事件结构化稳定性是核心未验证假设**
- 出处：[ARCHITECTURE §3.1, §7]
- 问题：3B 模型对复杂中文情境幻觉率经验值 15-25%，整个本地链路依赖此环节
- 处理：M0.2 必须先做 100 条人工标注样例的基线测试，准确率 < 85% 就上 7B 或退到云端结构化（仍按压缩控本）

**F8. 反馈"自动改 3 文件 100 行"边界过于粗糙**
- 出处：[FEEDBACK_LOOP §3]
- 处理：分类设阈值——A 类(文案) ≤ 1 文件 30 行；B 类(阈值) ≤ 1 文件 10 行；C/D 一律人工

**F9. 主动反馈按钮违背"零摩擦"差异化**
- 出处：[DIFFERENTIATION §3.2] vs [FEEDBACK_LOOP §1]
- 处理：改为被动反馈为主——是否打开周报、停留时长、是否复述给配偶；显式按钮只在"危机反馈通道"保留

**F10. 高置信度 / 主动洞察触发没有阈值定义**
- 出处：[ARCHITECTURE §4.2]
- 处理：在 ARCH §4.2 写明触发条件矩阵（confidence + 用户历史采纳率 + 是否曾被父母禁用该领域）

### 🟢 补缺（可以排到下一版）

**F11. 冷启动 prompt 过拟合瑶瑶**：第 4 周起加合成 3-5 个虚拟孩子画像做对照
**F12. Kill switch / 放弃条件**：16 周后若(a) 周打开 < 1 或(b) 月成本 > $10/用户 或(c) 信号准确率 < 60%，停止开发
**F13. 早期用户 GTM 草稿**：哪怕一页
**F14. 隐私威胁建模 (PRIVACY_THREAT_MODEL.md)**：STRIDE/LINDDUN 视角；离婚/家暴/孩子成年要求删除等场景
**F15. AI 失败的"危机反馈通道"**：UI 上加紧急按钮
**F16. 测试 fixtures 必须用合成数据**：CLAUDE.md §4.4 已说"瑶瑶 30 天"会让人误读为真实数据，明确改成虚构孩子"小明"
**F17. ROADMAP 状态列并发编辑**：拆出独立 ROADMAP_STATUS.md，避免 Cowork × Code 冲突
**F18. memory/ 对 Code 只读**：在 COLLABORATION 写明
**F19. sqlite-vec vs chromadb 现在就拍板**：选 sqlite-vec（嵌在 SQLite 里，零运维）
**F20. iPhone 端推理可行性 spike**：v1 之前先 benchmark，否则会推倒重来

## Decision (2026-05-17 用户裁决)

| 编号 | 用户决策 | 落地位置 |
|---|---|---|
| F1 | **修改采纳** — 采纳率保留为辅助参考，新增多元指标（讨论触发、自主新决定、被动信号），8 周后一起看哪个最有价值 | ROADMAP M3.3 |
| F2 | **延后** — 前期先有 APP，正式北极星 v0.2 再定义 | ROADMAP §0 |
| F3 | **采纳并强化** — MVP 假设降级为"重复模式 + 成长变化轨迹"；特别强调孩子发展是连续的、不能把信号当恒定特质 | ROADMAP 顶部、Phase 1 |
| F4 | **采纳轻量版** — README 加私人项目自用声明，v1 商业化前补合规 | README |
| F5 | **延后** — 后续采购服务器时再考虑长期保存 / 密钥继承 | backlog |
| F6-F20 | **暂不处理**（按用户"保守路径"原则） | backlog |

ADR 状态转 **accepted**。


## Consequences
不处理 F1–F5 = 项目从根基就有逻辑偏差，越写越远。
F6–F10 不处理 = MVP 阶段会反复打补丁。
F11–F20 不处理 = 短期可承受，但下一阶段会变阻碍。
